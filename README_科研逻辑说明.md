# 科研项目执行步骤与逻辑思维说明

本文件记录了本项目关于 Web of Science (WoS) 文献数据分析的整理逻辑与执行步骤，旨在遵循“知行合一”的科研方法论，确保研究过程的可追溯性与严谨性。

---

## 一、 逻辑思维框架：知行合一

本项目的组织结构基于科研工作的三个核心阶段：

1.  **知 (感知与获取)**：对应 `01_原始数据`。这是研究的起点，强调对客观事实（文献数据）的忠实记录与存储。
2.  **行 (实践与处理)**：对应 `02_计算与程序`。这是研究的核心过程，通过编写自动化工具（PowerShell 脚本）对原始数据进行清洗、提取和统计分析，将“数据”转化为“信息”。
3.  **果 (产出与验证)**：对应 `03_计算结果`。这是研究的终点，将处理后的信息转化为可视化的分析报告，用于支撑论文观点或进一步的研究决策。

---

## 二、 执行步骤记录

### 第一步：环境梳理与需求分析 (2025-12-25)
- **任务**：分析 WoS 导出的 `savedrecs.txt` 文件结构。
- **发现**：识别出关键字段如 `PT` (出版物类型)、`AU` (作者)、`TI` (标题)、`PY` (年份)、`TC` (被引频次)、`SO` (刊名) 及 `WC` (学科分类)。
- **逻辑**：确立了以“自动化提取”替代“人工统计”的原则，以提高准确性。

### 第二步：文件夹结构标准化
- **操作**：创建了 `01_原始数据`、`02_计算与程序`、`03_计算结果` 三级目录。
- **逻辑**：通过物理隔离，防止程序误删原始数据，同时使项目结构对外部合作者清晰易懂。

### 第三步：程序开发与调试
- **工具**：选择 PowerShell 作为处理工具（兼容 Windows 环境且无需额外安装库）。
- **核心逻辑**：
    - 使用正则表达式匹配 WoS 的两字母标签（如 `AU `）。
    - 处理跨行内容（如长标题的换行续接）。
    - 针对 `TC` (引文数) 字段进行数值化处理以支持降序排列。

### 第四步：数据处理与报告生成
- **执行**：运行 `analyze_data.ps1`。
- **产出**：自动在 `03_计算结果` 目录下生成《数据分析报告.txt》。
- **逻辑**：报告内容按“基础概况 -> 趋势分析 -> 核心载体 -> 学术影响力”的层级递进，符合科学汇报逻辑。

---

## 三、 维护与更新指南

1.  **更新数据**：若有新的 WoS 导出文件，请替换 `01_原始数据/savedrecs.txt`。
2.  **重新计算**：右键点击 `02_计算与程序/analyze_data.ps1` 选择“使用 PowerShell 运行”。
3.  **查看结果**：直接打开 `03_计算结果/数据分析报告.txt` 即可获取最新统计信息。
